# AGENTS.md

## Directrices de desarrollo

- Seguir los principios SOLID.
- Seguir una arquitectura orientada a Clean Architecture.
- Seguir las buenas prácticas de clean code.
- Seguir los lineamientos de OOP.
- Trabajar con TDD.

## Criterios de implementación

- Priorizar casos de uso pequeños, cohesionados y fáciles de testear.
- Mantener la lógica de dominio desacoplada de frameworks e infraestructura.
- Diseñar contratos claros entre capas antes de implementar adaptadores.
- Escribir o actualizar tests antes de cerrar cambios funcionales.
- Evitar soluciones mágicas o implícitas cuando perjudiquen legibilidad o mantenibilidad.
